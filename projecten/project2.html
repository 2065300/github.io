Aanleiding

Ik heb eigenlijk meerdere redenen voor een persoonlijke website. Laten we beginnen bij mijn fotografie redenen. Ik ben nu al 4 jaar geinteresseerd in fotografie en mijn foto’s waren niet echt publiekelijk beschikbaar. Ze zaten gewoon in een folder op mijn pc, virtueel stof te collecten. Door het maken van een website heb ik de mogelijkheid om deze foto’s naar de buitenwereld te presenteren. Het ziet er professioneler uit dan ze op instagram te plaatsen. Bovendien bereik ik hier meer mensen en is dit een manier om echt de focus te leggen op de fotografie.

Daarnaast komt een eigen website professioneel over en wil ik deze gebruiken om mijn projecten en leerervaringen op het gebied van ICT te documenteren/bekend te maken. Door het publiceren van mijn projecten kunnen geïnteresseerden mijn werk inzien en creëer ik voor mezelf een plek waar ik mijn vooruitgang in kennis kan zien.

De laatste reden waarom ik een eigen website wou is omdat het mij gewoon altijd aansprak, gewoon het idee al. Dacht alleen eerlijk gezegd niet dat het in 2023 al een realiteit zou worden. Hier vertel ik in een blog artikel meer over.

Wat heb ik geleerd

Door ervoor te kiezen om een website op een pc te hosten in plaats van een traditionele webhostingservice te gebruiken, heb ik kennis opgedaan over verschillende technische aspecten van webhosting. Bijvoorbeeld door te leren hoe Apache werkt als webserversoftware, die wordt gebruikt om webpagina's aan bezoekers te leveren. Bovendien heb ik meer met linux gewerkt wat ervoor zorgde dat ik een beter begrip kreeg van de filesystem, command-line en de netwerkprotocollen.

Het routeren van verkeer naar een pc vereist het instellen van port forwarding op de router, wat een betere begrip heeft opgeleverd van hoe netwerkverkeer wordt omgeleid naar specifieke apparaten op een lokaal netwerk. Bovendien heeft het proces van het instellen van een domeinnaam en het verbeterde van DNS-inzicht gegeven in hoe domeinnamen kunnen worden vertaald naar IP-adressen en hoe DNS-servers werken.

Ook heb ik me verdiept in hoe ik het veilig kan hosten, en hoe het gebruik van een reverse proxy mijn publieke ip adres kan voorbergen. Verder heb ik ook een goede basis kennis opgedaan over HTML CSS, javascript en SQL. Dit heb ik geleerd door video’s te kijken, maar vooral gewoon veel oefenen.

Requirements

Voor het maken van de website heb ik eerst een aantal requirements geformuleerd(eisen voor de website) Deze bestonden vooral uit de functionaliteiten die de website moet bevatten. Zo heb ik nagedacht over: Wat voor pagina’s wil ik? Wat moet de gebruiker kunnen doen? Wat wil ik weergeven? Deze vraagstukken hebben me geleid tot het maken van een lijst met verschillende requirements.

Ik heb ook goed nagedacht over het beheer van mijn website. Aangezien ik infrastructure/security erg interessant vindt, heb ik besloten om de server zelf te hosten op mijn eigen linux server. Bekijk deze link voor de opzet daarvan.

Door het beantwoorden van deze vraagstukken heb ik een duidelijker beeld gecreerd over het project. Het maken van de wireframes heeft hier ook zeer goed bij geholpen. Kom ik later op terug.

Hoe is mijn website ontwikkeld?

In dit artikel ga ik niet te technisch worden over het design. Ik begon met 0 kennis over html, CSS en javascript. Kunnen sommige onderdelen beter? Ja, mijn CSS files(deze zorgen voor het uiterlijk van de website) hebben waarschijnlijk veel onnodige attributen of fixed values, wat uiteraard beter kan. Maar is dat nodig? Niet echt, want ik ben blij met mijn website. Hij werkt en ik hoef me niet verder te verdiepen in front-end development aangezien ik niet in dit vakgebied wil werken, het is natuurlijk wel goed om wat basis kennis ervan op te doen.(als iemand een CSS expert is en heel graag feedback wilt geven op mijn files, neem contact met me op ;D)

Hoe heb ik dit gedaan?

Youtube videos gekeken, experimenteren en vooral veel gegoogled. De waves die je onderaan de pagina ziet, heb ik kunnen realiseren door gewoon een youtube video te kijken. Er is zoveel informatie gewoon beschikbaar. De waves leek me wel een grappig toevoeging, ik kreeg een beetje “Go with the flow” vibes.

Waar liep ik tegen aan?

het maken van de api die content van mijn database op mijn site zou weergeven. Het idee was om content van de artikelen en de foto’s vanuit de database weer te geven. Na veel onderzoek hoe flask werkt, en hoe de de javascript code eruit moest zien. Lukte het om lokaal data vanuit de database weer te geven. Deze api werkte via http. De rest van mijn site via https, dus dat zorgde voor mix content.

Ik had besloten om de api weg te halen en alle content er statisch in te zetten. Ik moest anders de api op een ander domein naam registreren en dat is het niet waard. Ondanks het dus niet lukte, heb ik hier wel veel van geleerd en kan ik het wel lokaal voor andere hobby projecten gebruiken.

Verder was het maken van de CSS files soms ook echt een gedoe, Ik heb soms veel tijd besteed aan het zorgen dat een bepaald element op de juiste positie stond. Of dat er veel attributen aangepast moeten worden omdat ze elkaar overschreden.

Hoe verliep het testen?

Elke nieuwe versie heb ik lokaal gedeployed, hierdoor kon ik het responsive design goed testen op mijn telefoon. Ook heb ik mensen gevraagd om te kijken naar het ontwerp om zo nieuwe inzichten te krijgen. User experience is een belangrijk onderdeel, dus ik kan niet alleen uitgaan op mijn eigen beeld.

Deze nieuwe inzichten hebben me vooral geholpen om het design en de functionaliteiten van de website te verbeteren. Een belangrijk punt wat me werd verteld was dat ik bijvoorbeeld de footer en navigatie balk moest weglaten als je op een foto klikt vanuit de fotografie pagina. Hierdoor ontstaat er echt meer focus op de foto. Tevens ziet het design er echt strakker uit. Thanks Beau.
